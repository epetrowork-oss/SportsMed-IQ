# SportMedIQ image brief — Batch 1 of ~6 (app chrome: hero, category icons, card thumbnails)

You are producing 26 WebP images for SportMedIQ, an offline-first learning
PWA for high school sports medicine (grades 7-12). This batch is the app
"chrome" — the home hero, 7 category icons, and 18 unit-card thumbnails.
Lesson diagrams come in later batches.

## Style guide (applies to every image in this batch)

- **Look**: flat, modern educational illustration; consistent line weight;
  friendly but not childish (audience spans 7th-12th grade). No photoreal,
  no gore, no gradients-heavy 3D.
- **Palette**: the app brand is deep teal `#0e7490` on soft cool grays —
  use teal as the recurring accent so the set feels like one family.
  Supporting colors should stay muted (the app uses soft category tints:
  red/orange/violet/blue/teal/green/slate).
- **People**: diverse high-school athletes and student athletic trainers,
  modern practice/sideline settings.
- **No text inside images** (every "Text in image" cell says None).
- **Format**: WebP. Icons/thumbnails ≤ 80 KB each, hero ≤ 150 KB.
- **Category icons**: simple flat glyph style, single-concept, readable at
  40px, transparent background — these sit inside small tinted tiles.
- **Thumbnails (3:2)**: one clear scene or object per unit topic, readable
  at card size (~300px wide), white background.
- **Exact filenames and folders matter** — the app looks for these paths
  under `public/images/`. Deliver each file to its Location column path
  with its Asset column filename, unchanged.
